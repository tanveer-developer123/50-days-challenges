SQLite Joins - Complete Guide
1Ô∏è‚É£ INNER JOIN

    Definition: Sirf un rows ko return karta hai jinke values dono tables me match karte ho.

    Syntax:

SELECT columns
FROM table1
INNER JOIN table2
ON table1.column = table2.column;

    Example:

SELECT users.name, posts.title
FROM users
INNER JOIN posts
ON users.id = posts.user_id;

    Explanation: Sirf un users ke posts dikhaye jate hain jinke posts exist karte hain.

2Ô∏è‚É£ LEFT JOIN

    Definition: Left table ke saare rows dikhaye aur match na hone par NULL fill kare.

    Syntax:

SELECT columns
FROM table1
LEFT JOIN table2
ON table1.column = table2.column;

    Example:

SELECT users.name, posts.title
FROM users
LEFT JOIN posts
ON users.id = posts.user_id;

    Explanation: Sab users show honge, chahe posts ho ya na ho.

3Ô∏è‚É£ RIGHT JOIN

    Definition: Right table ke saare rows dikhaye aur match na hone par NULL fill kare.

    Note: SQLite me direct RIGHT JOIN nahi hota, LEFT JOIN ke saath table order swap karke kar sakte hain.

    Example:

SELECT users.name, posts.title
FROM users
RIGHT JOIN posts
ON users.id = posts.user_id;

    Explanation: Sab posts show honge, chahe unka user exist na kare.

4Ô∏è‚É£ FULL OUTER JOIN

    Definition: Dono tables ke saare rows, match na hone par NULL fill kare.

    SQLite workaround: UNION of LEFT JOIN and RIGHT JOIN

SELECT users.name, posts.title
FROM users
LEFT JOIN posts
ON users.id = posts.user_id
UNION
SELECT users.name, posts.title
FROM users
RIGHT JOIN posts
ON users.id = posts.user_id;

5Ô∏è‚É£ CROSS JOIN

    Definition: Cartesian product, sab combinations.

    Syntax:

SELECT columns
FROM table1
CROSS JOIN table2;

    Example:

SELECT users.name, posts.title
FROM users
CROSS JOIN posts;

    Explanation: Har user ke saath har post dikhaye jaayenge.

6Ô∏è‚É£ SELF JOIN

    Definition: Table ko apne aap se join karna. Hierarchies ke liye.

    Syntax:

SELECT e.name AS employee, m.name AS manager
FROM employees e
LEFT JOIN employees m
ON e.manager_id = m.id;

    Example Output:
    | Employee | Manager |
    |----------|---------|
    | Ali | NULL |
    | Ahmed | Ali |
    | Sara | Ali |
    | Bilal | Ahmed |
    | Ayesha | Ahmed |
    | Usman | Sara |

    Explanation: Employee aur manager ke names ek hi table se nikale gaye.

Join Type	Use Case Example
INNER JOIN	Users with existing posts
LEFT JOIN	All users + their posts (null if none)
RIGHT JOIN	All posts + their users (null if no user)
FULL JOIN	Complete list from both tables
CROSS JOIN	Combinations, cartesian products
SELF JOIN	Hierarchy / employee-manager / tree structures
